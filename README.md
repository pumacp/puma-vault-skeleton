> A generic, reusable, **self-documenting** knowledge-organization vault for Obsidian — and any Markdown tool. It helps you turn complex, abstract, fuzzy information into high-quality structured knowledge, and it explains its own workings *using its own methods*.

## Who it is for
Students · teachers · researchers · artists · programmers · information managers · writers · PhD candidates — anyone who must organize a lot of complex knowledge and wants a principled, ready-made structure instead of a blank vault.

## What you get
A complete folder skeleton on a **Johnny Decimal** spine, pre-wired with seven fused core methods, opt-in modules, worked examples in every area, note-type templates, and Maps of Content — all in plain Markdown with **zero lock-in**.

| Layer | Method | Role |
|-------|--------|------|
| Navigation | [Map of Content](60%20-%20Resources/67%20Methodologies/67.1%20Core/Map%20of%20Content%20(MOC).md) | orientation maps |
| Capture | [GTD](60%20-%20Resources/67%20Methodologies/67.1%20Core/GTD.md) | tasks & inbox |
| Organize | [PARA](60%20-%20Resources/67%20Methodologies/67.1%20Core/PARA.md) + [Johnny Decimal](60%20-%20Resources/67%20Methodologies/67.1%20Core/Johnny%20Decimal.md) | folders & addresses |
| Distill | [Zettelkasten](60%20-%20Resources/67%20Methodologies/67.1%20Core/Zettelkasten.md) | atomic linked notes |
| Express | [LLM Wiki](60%20-%20Resources/67%20Methodologies/67.1%20Core/LLM%20Wiki.md) | publish / AI context |
| Orchestration | [CeReBro (CODE)](60%20-%20Resources/67%20Methodologies/67.1%20Core/CeReBro%20(CODE).md) | binds all of the above |

Opt-in modules (research, development, prompting, learning) and clearly-fenced *reference-only* methods are documented under `60 - Resources/67 Methodologies/`.

## Structure (Johnny Decimal root)
```
00 Meta · 10 Inbox · 20 Literature · 30 Permanent · 40 Projects
50 Areas · 60 Resources · 70 Archive · 80 MOC · 90 GTD
```

## Get started in minutes
1. **Clone or download** this repository.
2. **Obsidian:** *Open folder as vault* → select the folder. (Obsidian writes its own config on first open; none ships here.)
3. Open `index.md` or `00 - Home.md` and follow `MOC Navigation`.
4. Copy `40 - Projects/_EXAMPLE-Project` to start your first project.

It also works in **Logseq, Foam, VS Code, or by simply reading the files**. Full guide: [`Getting-Started.md`](Getting-Started.md).

## Publish a website (optional)
Turn the vault into a site with **Quartz 4** (jackyzha0). No Quartz config ships here — see [`Quartz-Publishing.md`](Quartz-Publishing.md).

## Quality
The skeleton targets: 0 broken links · ≥7 core methods each with ≥2 worked examples · 100% of explanatory notes classified. How to verify it yourself (no code shipped): [`Verification.md`](Verification.md).
