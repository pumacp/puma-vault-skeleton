# 🚀 Getting Started

This skeleton is **plain Markdown** — a folder of `.md` files. It works in Obsidian, Logseq, Foam, VS Code, or just by reading the files. No application config ships with it, so you stay in full control.

## Option A — Obsidian (recommended for the full experience)
1. Install [Obsidian](https://obsidian.md).
2. **Open folder as vault:** *Open* → *Open folder as vault* → select this repository's folder.
3. Obsidian creates a fresh `.obsidian/` config with sensible defaults on first open. Nothing here overwrites your preferences.
4. Open [[00 - Home]] to start. Follow [[MOC Navigation]] to explore.
5. (Optional) Enable community plugins you like — the vault assumes none, but pairs well with a Dataview-style query plugin and a spaced-repetition plugin (see [[Spaced Repetition]]).

## Option B — Logseq / Foam / VS Code
Open the folder directly. `[[Wikilinks]]` work in Logseq and Foam; in VS Code use any Markdown/wikilink extension. The structure and content are identical.

## Option C — Tool-free
Just read the Markdown. The Johnny Decimal numbers ([[JD-Master-Index]]) tell you where everything is, and every note ends with a **See also** section so you can follow links by hand.

## First 10 minutes
1. Read [[index]] and [[00 - Home]].
2. Skim [[README-Methodologies]] and [[MOC-Methodologies]] to learn the seven core methods.
3. Look at a worked example pair, e.g. [[EX-Project-Charter]] ↔ [[EX-Project-NextActions]].
4. Copy `40 - Projects/_EXAMPLE-Project` to start your own project.

## Make it yours
- Capture into `10 - Inbox`, then file by actionability with [[PARA]].
- Give each new note a numeric address with [[Johnny Decimal]].
- Distill sources into atomic notes with [[Zettelkasten]]; connect them with a [[Map of Content (MOC)]].

To publish your vault as a website, see [[Quartz-Publishing]].
