# CeReBro (CODE)

> The orchestration cycle that connects every other core method: **Capture · Organize · Distill · Express**. "CeReBro" is simply the Spanish-language framing of this same Building-a-Second-Brain workflow.

## How it works
CODE is a four-phase loop that moves information from raw input to shared output:

1. **Capture** — keep what resonates (front door: GTD inbox).
2. **Organize** — file it by actionability (PARA) at a stable address (Johnny Decimal).
3. **Distill** — extract the essence into atomic notes (Zettelkasten), mapped by MOCs.
4. **Express** — turn knowledge into output you share (LLM Wiki, permanent notes, publishing).

> [!note] On the name "CeReBro"
> *CeReBro* (from Spanish *cerebro*, "brain") is a Spanish denomination of Forte's **Building a Second Brain / CODE**. It is **not** an independent method. Where older material used the Spanish phase labels *Captacion / Organizacion / Brainstorming / Realizacion*, this skeleton standardizes on the canonical **Capture / Organize / Distill / Express**, with the mapping **Brainstorming ≈ Distill** and **Realizacion ≈ Express**.

## Worked examples
- **Example 1 — One idea, end to end.** A talk you hear is *captured* to the inbox, *organized* as a Resource under JD `60`, *distilled* into a permanent note linked from a MOC, and *expressed* as a published article — traceable across all four folders.
- **Example 2 — A Distill pass.** You revisit a literature note and apply progressive summarization (bold the key lines, then summarize them) until a single reusable permanent note remains. See [[Progressive Summarization]].
