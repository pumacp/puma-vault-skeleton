# LATCH

> Richard Saul Wurman's claim that there are only **five ways to organize any information: Location, Alphabet, Time, Category, Hierarchy.**

## How it works
Any collection can be arranged along one of five axes:

- **Location** — by place or spatial relationship.
- **Alphabet** — A–Z, for large lookup sets.
- **Time** — chronological or sequential.
- **Category** — by kind or theme.
- **Hierarchy** — by magnitude or importance.

The point is to choose the axis that fits the reader's need.

## Source
Richard Saul Wurman, *Information Anxiety* (1989).

## Why not integrated
LATCH **competes with [[PARA]] + [[Johnny Decimal]] in the same organizational layer**, so adopting it would mean replacing the core, not extending it. Its genuinely distinct axes are already covered by the vault through other means:

- *Location / Category / Hierarchy* → handled by PARA + the JD folder tree.
- *Time* → handled by note frontmatter dates and chronological queries.
- *Alphabet* → handled by file managers and search.

So LATCH adds no **bridge** the core does not already provide. It is worth knowing as an information-architecture lens, but it is **not implemented** here.
