# DIKW

> The pyramid that arranges **Data → Information → Knowledge → Wisdom** as ascending levels of meaning and usefulness.

## How it works
Each level refines the one below:

- **Data** — raw symbols, no context.
- **Information** — data with context ("who, what, where, when").
- **Knowledge** — information organized for use ("how").
- **Wisdom** — judgment about what is worth doing ("why").

## Source
Commonly attributed via Russell Ackoff, "From Data to Wisdom," *Journal of Applied Systems Analysis* (1989).

## Why not integrated
DIKW is a **conceptual frame, not an operational method**. It describes a *direction* of refinement that the core already enacts (the [[CeReBro (CODE)]] Distill phase moves captured data toward usable knowledge), but it provides **no concrete workflow, file convention, or practice** to adopt. There is nothing to *do* with DIKW inside the vault beyond the metaphor, so it offers no bridge to implement. Useful as background vocabulary; **not implemented**.
