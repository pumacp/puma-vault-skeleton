# Diataxis

> A framework that splits documentation into four distinct modes — **Tutorials, How-to guides, Reference, and Explanation** — because each serves a different user need.

## How it works
Each mode answers a different question and should not be mixed:

| Mode | Serves | Question |
|------|--------|----------|
| Tutorial | learning | "teach me by doing" |
| How-to | a task | "how do I achieve X?" |
| Reference | lookup | "what exactly is X?" |
| Explanation | understanding | "why is it this way?" |

## Source
Daniele Procida, "Diataxis" documentation framework (diataxis.fr).

## Bridge
Structures the skeleton's **self-documentation** and the published site: the [[LLM Wiki]] / Quartz output is organized into the four modes, realizing the **Express** phase of [[CeReBro (CODE)]]. Opt in when you publish docs or a wiki.

## Worked examples
- **Example 1 — Splitting a README.** An overloaded README is divided into a *tutorial* (Getting Started), *how-to* (Quartz publishing), *reference* (the JD index), and *explanation* (the methodology notes).
- **Example 2 — Wiki pages.** The published Wiki uses one page per mode per topic, so a beginner and an expert each find the page that fits their need.
